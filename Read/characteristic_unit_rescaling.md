`model.py` restores a realistic metric-like distance scale after `dataset.py` has normalized each scene into a unit sphere. In the active path, `dataset.py` first centers the scene and divides all coordinates by the scene radius, so point spacing is no longer in raw MultiScan units. Then `model.py` estimates the normalized scene's local point spacing with a nearest-neighbor statistic, using the configured spacing metric (`p05` here) over a random point sample. If the measured spacing is `s` and the target characteristic unit from the MultiScan survey is `t = 0.013616`, `model.py` rescales every point coordinate by `t / s`, so the scene's local spacing matches the surveyed MultiScan density again. After that, it applies the standard SoftGroup test-time rotation and multiplies coordinates by the voxel scale only for sparse voxel indexing, while SoftGroup's grouping and neighbor queries still use the restored floating-point coordinates. In short, the algorithm does not recover absolute room size; it restores the characteristic local sampling distance so SoftGroup's metric-sensitive radius and voxel logic behave as if the point cloud were back in the intended MultiScan distance regime.
