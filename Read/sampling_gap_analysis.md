## 측정 항목 설명

### Spacing (점 간격)
한 씬 또는 한 인스턴스 내의 포인트들이 얼마나 촘촘하게 분포하는지를 나타내는 지표.

계산 방법:
1. 포인트 중 최대 3,000개를 랜덤 샘플링
2. 각 점에서 **가장 가까운 이웃 점까지의 거리** (KNN k=2)를 계산
3. 그 거리들의 **중앙값(median)** = spacing

값이 작을수록 점들이 더 촘촘하게 분포.

### 인스턴스당 점 수
씬 안에서 하나의 넙죽이 객체(instance_label == i, i>0)에 속하는 포인트의 개수.
하나의 씬에 넙죽이가 3개 있으면 3개의 값이 나오고, 이를 전체 씬에 걸쳐 평균낸다.

## 핵심 발견

### 1. 배경 밀도는 동일

| | 배경 spacing (median) |
|---|---|
| 조교 파일 (6개 평균) | 0.0174 |
| 우리 파일 (174개 평균) | 0.0175 |

배경은 동일한 MultiScan OIS 씬에서 왔기 때문에 밀도가 거의 동일하다.
**배경은 문제가 없다.**

### 2. 넙죽이 밀도가 크게 다르다

| 항목 | 조교 (5개 고유 씬) | 우리 (174개 씬) | 배율 |
|---|---|---|---|
| 인스턴스당 점 수 (평균) | 14,969 | 2,479 | **6배** |
| 인스턴스당 점 수 (중앙값) | 15,928 | 1,536 | **10배** |
| 넙죽이 spacing (씬 단위 median) | 0.0018 | 0.0093 | **5배** |
| 넙죽이 / 배경 spacing 비율 | **0.10** | **0.53** | — |
| 씬당 인스턴스 수 (평균) | 4.3 | 2.9 | 1.5배 |

### 3. 조교 인스턴스별 상세 spacing

조교 파일을 인스턴스 단위로 보면:

| 파일 | 인스턴스 | 점 수 | spacing |
|---|---|---|---|
| test_case_000 | inst1~5 | 12,430~18,593 | 0.0012~0.0038 |
| test_case_001 | inst1~4 | 2,696~25,036 | 0.0009~0.0034 |
| test_case_002 | inst1~4 | 4,999~23,264 | 0.0012~0.0032 |
| test_case_003 | inst1~3 | 2,557~16,026 | 0.0020~0.0033 |
| test_case_004 | inst1~5 | 7,974~26,924 | 0.0010~0.0048 |

전체 범위: spacing **0.0009~0.0048**, 점 수 **2,557~26,924**

spacing이 인스턴스마다 다른 것은 각 넙죽이가 다른 스케일로 배치되었기 때문으로 추정.
작은 스케일로 배치된 경우 점들이 더 촘촘하고, 크게 배치된 경우 더 성기다.
(씬 좌표계에서의 절대 거리가 스케일에 비례하기 때문)

## 근본 원인

현재 `generate_synthetic_dataset.py`의 `_sample_object_points()`:

```python
OBJECT_POINT_SPACING_RATIO_RANGE = (0.85, 1.10)
target_spacing = scene_point_spacing * spacing_ratio  # 배경과 거의 같은 밀도
```

넙죽이 샘플링 간격이 배경 간격의 85~110%로 설정되어 있어,
넙죽이가 배경과 거의 같은 밀도로 샘플링된다.

조교 데이터는 넙죽이 spacing이 배경의 약 10% 수준이고,
절대값 기준으로 **0.001~0.005** 범위에 분포한다.

## 수정 계획

### 수정 1: scene_point_spacing 기반 비율 방식 → 절대 spacing 방식으로 전환

**현재:**
```python
OBJECT_POINT_SPACING_RATIO_RANGE = (0.85, 1.10)
target_spacing = scene_point_spacing * spacing_ratio
```

**수정안:**
```python
OBJECT_FIXED_SPACING_RANGE = (0.001, 0.004)  # 절대 거리값 (scene 좌표계)
target_spacing = float(rng.uniform(OBJECT_FIXED_SPACING_RANGE[0], OBJECT_FIXED_SPACING_RANGE[1]))
```

근거:
- 조교 인스턴스별 spacing 범위가 0.0009~0.0048 → 절대값 기준으로 0.001~0.005에 집중
- 배경 spacing(0.0174)과 무관하게 고정 고밀도로 샘플링하는 것이 조교 방식과 일치
- 비율 방식은 배경 밀도에 종속되어 도메인 갭을 만들지만, 절대값 방식은 항상 일정한 품질 보장

예상 효과:
- 인스턴스당 점 수: 2,479 → ~10,000~20,000
- POINT_COUNT_RANGE 상한(32,768) 이내로 수용 가능

### 수정 2: 씬당 인스턴스 수 상향

조교 평균 4.3개 vs 우리 평균 2.9개.
`MIN_INSERTIONS`를 2~3으로 높이는 것 검토.

### 수정 3: 재생성 후 재검증

수정 후 동일 스크립트로 측정하여 아래 기준 충족 확인:
- 인스턴스당 점 수 평균 >= 10,000
- 넙죽이 spacing 범위 0.001~0.005 내 분포
